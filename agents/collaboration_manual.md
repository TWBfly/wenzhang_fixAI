# 🤝 IDE Agent 协作指南：短篇小说深度润色流程

本指南旨在指导 IDE 中的 Agent (如 Antigravity) 如何与 `duanpian_fixAI_1.py` 脚本协作，完成从“机器初改”到“文学级润色”的闭环。

## 1. 协作模型
- **脚本 (Script)**：负责高频词过滤、逻辑审计、节奏分析及数学公式拦截。输出基础重写版及 [审计任务]。
- **Agent (You)**：负责感性认知、修辞优化、语境补全及“拟人化”润色。

## 2. 操作流程

### 第一步：获取待办任务
脚本运行结束后，会在 `wenzhang_fixAI/agent_tasks/` 目录下生成 `refine_[章节名].md` 文件。请定期检查该目录。

### 第二步：解析任务指令
打开任务文件，您会看到：
- **Audit Findings**：脚本拦截到的硬伤（如：严重的 AI similes）。
- **Self-Check List**：必须遵守的文学准则。
- **Source Text**：为您准备好的基础稿。

### 第三步：执行深度润色
根据任务要求，在 IDE 中直接生成优化后的文本。
- **严禁**：使用任何外部 API Key 或 Local LLM Server。
- **必须**：使用您自身的模型推理能力进行文学创作。

### 第四步：同步结果
将润色后的文本保存回原文件路径，或在对话中提交给用户。

## 3. 文学准则 (Core Principles)
- **Show, Don't Tell**：将“他很生气”改为“他重重地合上了书”。
- **去模板化**：删除“心脏狂跳”、“瞳孔收缩”等 AI 比喻。
- **去逻辑词**：删除“这意味着”、“事实上”、“显然”。
- **节奏化**：长短句错落，增加段落间的呼吸感。
